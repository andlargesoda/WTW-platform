Онлайн-кинотеатр "Что посмотреть?"
Новое ИТ-решение
1 Бизнес-цель проекта: 
1.1 Увеличение трафика из поисковых систем
1.2 Увеличение аудитории
1.3 Повышении вовлеченности
1.4 Размещение рекламы для монетизации
Целевая аудитория: Люди от 18 до 45, активно пользующиеся онлайн кинотеатрами
Аналог: Кинопоиск

2 ФТ:
2.1 Роли и пользователи
2.1.1 Незарегистрированный пользователь (гость)
2.1.2 Зарегистрированный пользователь
2.1.3 Администратор
2.2 Регистрация и авторизация
2.2.1 Варианты регистрации:
2.2.1.1 гостевой режим (регистрация не требуется, гостевой режим)
2.2.1.2 социальные сети: vk, одноклассники, ТГ (предпочтительный вариант)
2.2.1.3 e-mail (логин пароль)
2.2.2 Действия при регистрации
2.2.2.1 Подтверждение почты
2.2.2.2 После регистрации пользователь автоматически авторизируется в личный кабинет
2.2.2.3 При ошибке авторизации отображается ошибка
2.3 Управление списками фильмов
Дату просмотра хранить не требуется
Требуется подтверждающее действие при поиске
Пользователь может осуществлять поиск фильмов через встроенный поиск, который получает с основного сайта
Пользователь может добавлять фильм в любую подборку
Пользователь может добавлять название к подборке
Пользователь может добавлять фильм в подборку в один клик
Пользователь не может перемещать фильм между подборками
Пользователь может удалять фильм из подборки
Пользователь может добавить один фильм одновременно в несколько подборок
2.4 Подборки (публичные и приватные)
Незарегистрированный пользователь может просматривать публичные подборки
Зарегистрированный пользователь может создавать неограниченное число публичных подборок (доступна всем пользователям платформы, включая гостей)
Зарегистрированный пользователь может создавать неограниченное число приватных подборок (доступна только авторизированному автору)
Зарегистрированный пользователь может менять тип (публичный, приватный) подборки
Зарегистрированный пользователь может менять название подборки
Зарегистрированный пользователь может удалить подборку
2.5 Заметки к фильмам
Пользователь может добавлять текстовую заметку не более 5000 символов в любой момент
Пользователь может добавлять заметку в публичную и приватную подборку
Пользователь может редактировать заметку
Пользователь может удалять заметку
Пользователь не может форматировать текст заметки
2.6 Случайный выбор фильма
Пользователь может выбрать случайный фильм из подной выбранной подборки путем нажатия кнопки неограниченное кол-во раз
Просмотренные фильма из списка не исключаются
2.7 Просмотр публичных подборок
Роли:
Незарегистрированный пользователь и зарегистрированный пользователь
Пользователь может находить публичные подборки по поиску, используя фильтр (Название, Автор, Тег)
Пользователь может выбирать случайный фильм в публичной подборке
Пользователь может оставлять заметку в публичной подборке
Пользователь может добавлять публичные подборки в Избранное
Пользователь не может сохранять чужую публчиную подборку к себе
Пользователь не может комментировать публичные подборки
2.8 Поиск и фильтрация фильмов
Пользователь может фильтровать фильмы при поиске по жанру, названию, году выпуска, рейтингу, актеру
Пользователь может сортировать фильмы по дате добавления
2.9 Профиль пользователя
Пользователь может видеть имя и почту профиля
Пользователь не может просматривать статистику (кол-во просмотренных фильмов, активность)
Профиль пользователя является приватным
2.10 Администрирование
Администратор может контролировать пользовательский контент
Администратор может модерировать публичные подборки
Адинистратор имеет админ-панель
Администратор может удалять подборки
Администратор может предупреждать пользователя при нарушении правил
Администратор может блокировать пользователя при нарушении правил
2.11 Альтернативные и исключительные сценарии
Система должна отображать сообщение для пользователей, если платформа не доступна
Система должна деактивировать аккаунт при его удалении

3 НФТ
3.1 Производительность
Планируемое кол-во пользователей на старте проекта: 5 000 пользователей
Пиковые нагрузки не ожидаются
Скорость отклика системы критична
Система должна загружать страницу до 1 сек
Система должна поддерживать одновременную работу до 10 000 пользователей
При росте нагрузке требуется автоскейлинг
3.2 Доступность
Система должна работать 24/7
Техническое обслуживание системы должно составлять не более 1% от общего времени в год
Требуемые уровень доступности системы должен составлять 99%
Допустимы ночные регламентные работы
3.3 Безопасность
Требования к защите персональных данных согласно законодательству РФ в области защиты ПДн
Двухфакторная аутентификация не требуется
Требуется шифрование ПДн в базе
Ограничение кол-ва попыток входа в систему не требуется
3.4 Масштабируемость
Система должна поддерживать горизонтальное масштабирование
Система не должна поддерживать мультиязычность
3.5 Удобство использования
Целевая аудитория: Люди от 18 до 45, активно пользующиеся онлайн кинотеатрами
Интерфейс системы должен быть удобным, понятным, лаконичным, адаптивным
Требуется мобильная версия
Не требуется поддержка темной темы
3.6 Надежность и отказоустойчивость
Требуется ежедневное резервное копирование данных
Требуется план аварийного восстановления (Disaster Recovery)
3.7 Поддержка и сопровождение
Команда разработки платформы будет заниматься ее сопровождением
Частое добавление нового функционала не требуется
Документирование API требуется
Админ панель для управления контентом требуется
3.8 Логирование и мониторинг
Требуется логирование всех необходимых событий (входы в систему, создание подборок, ошибки)
Не требуется аналитики действий пользователя
Требуется централизованный мониторинг системы
3.9 Совместимость
Требуется совместимость с веб-браузерами ПК, ноутбуков, планшетов, мобильных телефонов
Требуется поддержка только современных браузеров (не старше 3 лет)
Требуется поддержка iOS и Android

4 Классы
4.1 Основные классы
Система должна хранить следующие данные о фильме/сериале: название, жанр, год выпуска, длительность, режиссер, актеры, постер, описание, рейтинг
Система должна хранить следующие данные о пользователе: Имя, почта, пароль, роль, статус (активен, заблокирован)
Подборка-некий набор ссылок на фильмы на основном сайте, объединенных единым названием
История действий пользователя не требуется хранить
4.3 Класс Фильм/Сериал
см. 4.1
Один фильм может иметь несколько жанров
4.4 Класс Подборка
Система должна хранить следующие данные о подборке: Название, Автор, состав подборки, дата создания, дата изменения, тип (обязательные), описание
Подборка принадлежит одному пользователю
4.5 Класс Заметка
Система должна хранить следующие данные о заметке: Автор, дата, текст, привязка к подборке (обязательные)
Один фильм может имет одну заметку
4.6 Связи между классами
Один фильм может находиться в нескольких подборках
Пользователь может создавать несколько подборок
Связь «Пользователь - Подборка» Один ко многим
Связь «Подборка - Фильм» - Один ко многим
При удалении пользователя данные архивируются
4.7 Ограничения
Ограничения по кол-ву подборок отсутствует
Ограничения по количеству фильмов в подборке отсутствует
Пользователь не может создавать подборки с одинковым названием
4.8 Наследование
Логика работы классов фильм и сериалы не отличается
Администратор наследник класса пользователь
4.9 Жизненный цикл объектов
При удалении фильма он пропадает из подборок
При изменении типа подборки с приватной на публичную она доступна всем
При изменении типа подборки с публичной на приватную она доступна автору
Историю изменения хранить не требуется
При удалении подборки удаляются связи
Не требуется soft delete (логическое удаление)
4.10 Дополнительные классы
Теги не требуются
Система рейтингов не требуется
Комментарии к подборкам не требуются
